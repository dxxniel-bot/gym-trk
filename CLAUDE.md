## What it is
A mobile-first web app (works on iPhone via "Add to Home Screen") to log gym workouts + nutrition. Rebuilt from the user's old Expo Go app (`ironLOG`). Aesthetic: **terminal/CLI, monochrome, JetBrains Mono**.

## Architecture
- **`index.html`** ŌĆö the ENTIRE app (HTML + CSS + JS inline). Single-page, JS-rendered screens via a `render()` router on `state.screen`. ~all logic here.
- **`manifest.json` + `sw.js`** ŌĆö PWA (installable + offline). `sw.js` is **network-first** for same-origin (online = latest deploy, offline = cache) ŌĆö cache name bump (`gymtrk-vN`) forces refresh.
- **`*-prototype.html`** ŌĆö static design references (superseded by `index.html`; keep for design history).

### Data model (localStorage key `gymtrk_db_v1`)
```
{ version, profile:{username,sex,age,weightKg,heightCm,activity,goal, since:'YYYY-MM-DD'(frozen first-use date, set in migrate)},
  settings:{unit, restSec(default 90), restBeep(default true), goals:{kcal,protein,carbs,fat,sodium,potassium,sugar,fiber,water,caffeine,alcohol,sleep(h, default 8)}},
  activeGym, gyms[], rotIdx,
  sleep:{ 'YYYY-MM-DD': hours },          // date = the morning you woke up; logged via openSleepLog(), counted in dbMass()
  steps:{ 'YYYY-MM-DD': n }, bodyweight:{ 'YYYY-MM-DD': kg }, stack:[...], goalHist:{...}, mealTags:[...], lastUsed:{...},
  machines:{ '<gymName>':{ '<exerciseNameLower>':{brand,setup,na} } },   // per-gym machine/brand/setup linking; falls back to split's machine default
  split:{ name, days:[ {id,name,tag,metric,allowFailure, exercises:[{name,muscle,type,machine,unilateral,sets(=defaultcount)}]} ] },
  sessions:[ {id,date,dayId,dayName,gym,type,exercises:[{name,muscle,machine,setup,sets:[{w,r,rir,unit,side,fsOn,extraW,isDrop,done}]}]} ],
  meals:{ 'YYYY-MM-DD':[ {id,time,tag,name,qty, kcal,protein,carbs,fat,sugar,sodium,potassium,fiber,caffeine,water} ] },
  foods:[ {id,barcode,name,brand,serving, per:{...same nutrients per 1 serving}} ] }
```
- **No-data-loss rule:** keep the LS key stable, never clear it, migrate additively in `migrate()`. Editing code never touches LS. `[settings]` has export/import JSON (backup) + "log in" restores from a pasted backup.

### Screens (state.screen)
`landing` ŌåÆ `onboard` ŌåÆ `home` (gym: rotation w/ ŌĆ╣ŌĆ║day picker, //NEXT, start/rest/skip, log-later, **//VOLUME** weekly-sets-per-muscle vs MEV/MAV/MRV, //STATS) ┬Ę `workout` (live logging + bg duration; log-later mode = backdated + date/dur + advance-rotation toggle) ┬Ę `macros` (kcal ring + macro rings w/ gŌćä% & loggedŌćäremaining toggles + intake bars + bloat + food log w/ per-item P/C/F + per-meal totals + tap-to-detail) ┬Ę `splitedit` (rename/add/remove days & exercises, edit exercise) ┬Ę `history` (past sessions ŌåÆ view/edit/delete) ┬Ę `settings` (//PROFILE editable, //TRAINING, //DATA export/import, rest-timer secs + beep toggle) ┬Ę `progress` (REAL: weekly-consistency dots + tiles peso/volumen/e1RM/pasos/agua/sue├▒o/kcal with lineCharts + Apple-Health-style drill-downs via `metricSeries(key,days)`/`openMetricDetail`) ┬Ę `stack` (supps/skincare/meds with schedules).
**Rest timer (v139):** auto-starts when a set is marked done (`done` handler; skipped for `loggedAfter` sessions). State = `db.activeWork.restEnd` (epoch ms, persisted by `save()` ŌĆö survives re-renders/reloads/iOS kills). UI = `#resttimer` capsule in static HTML BETWEEN `#view` and `#wfooter` (outside #view so `render()` never destroys it); a single global 500ms interval writes `textContent` only (`updateRestBar()`, never re-renders); `visibilitychange` recomputes from `Date.now()` (iOS freezes timers); expiry >30s late (app was killed) skips the notify. Audio = Web Audio chirp, `AudioContext` lazily created on the `done` tap (iOS gesture requirement) + `navigator.vibrate` for Android. Config: `settings.restSec/restBeep`.
**Overlays (v141/v142):** `showOverlay(html,{ms,onDone})` full-frame `.bootov`; `bootScreen()` = 1.3s terminal status on open (uptime/last session/weekly vol ╬ö/bw/sleep; once per app-open via `sessionStorage.gymtrk_boot`; only with username + Ōēź1 session; tap skips) ŌåÆ chains `snapRecap()` = once-daily day summary after 21h (kcal vs goal/pasos/sesi├│n/agua/sue├▒o; `localStorage.gymtrk_recap`=date; `?recap=1` forces for QA). Home shows `uptime` (days since `profile.since`) instead of streak ŌĆö streak stays in progress. No FOMO/confetti by design.
**Screen-change fade (v145):** `go()` sets `state._trans` only when the screen actually changes; `render()` re-triggers `.fadein` (140ms) ŌĆö NEVER on same-screen re-renders (every keystroke re-renders).
Gym stats helpers: `weeklySets(days)`, `VOL_LANDMARKS` (RP MEV/MAV/MRV), `volStatus()`, `isHardSet()` (RIRŌēż4), `e1rmTrend()` (Epley; surfaced in `progress`). `topLift()` is dead code (removed from UI, fn remains). Top-lift & raw-volume removed on purpose.
**Workout prefill (v68):** new sessions pre-fill each set's weight/reps/unit from the last logged session of that exercise (`lastLoggedSets`), flagged **`_pf`** (a suggestion; RIR not copied). `_pf` clears on edit (`setField`) or mark-done; `saveSession`'s filter (`(s.w||s.r)&&!s._pf`) and its `has` check drop still-`_pf` sets, so suggestions never save or count in stats until you confirm them (edit or Ō£ō). `[+ set]` duplicates the previous set (also `_pf`). `emptySet(ex,side,isDrop,src)` ŌĆö pass `src` to prefill.
Native-feel: viewport `maximum-scale=1,user-scalable=no`, body `position:fixed`, inputs `font-size:16px`, frame fills screen `@media(max-width:440px)`.

### Built-in base foods (`BASE_FOODS`)
A curated, code-embedded table (~259 entries) of generic **whole/basic foods** OpenFoodFacts lacks (huevo, res, cebolla, tortilla, frijol, aguacate, pechuga, arroz, caf├®ŌĆ”). Mexican-staple + protein-leaning, broad coverage (dairy, poultry, beef/pork, fish/seafood, legumes, grains, veg, fruit, nuts/seeds, fats, sugars/condiments, snacks, supplements, beverages, prepared dishes). Each is already in canonical shape (`per100` for all MACROKEYS, `base:'g'|'ml'`, `sizes` with natural piece/serving). Built via a terse `add(name,kcal,p,c,f,opt)` helper (`opt`: `s`ugar `fi`ber `na`+`k` mg `caf` `sv`/`pc`/`ct` grams `sl` label `base` `en`=English alias). Values are ~per-100 g edible-portion USDA approximations (composite dishes are rougher). **Bilingual search** (user reported better MFP hits in English): every entry has an `en` alias and `bfNorm()` (accent-insensitive, lowercased) matches **name OR alt** ŌĆö so "chicken"ŌåÆPechuga de pollo, "onion"ŌåÆCebolla. **Surfaced in search, not copied to LS:** matched in `openFoodAdd`'s inline `#fa_q` list (under a B├üSICOS group) and in `openNameSearch` (B├üSICOS above OFF PRODUCTOS). Pick ŌåÆ `data-act="pickbase"` ŌåÆ `openLog(food,{isNew:true})`; "guardar + loguear" copies it into `db.foods` (keeps its `bf_*` id, then shows under RECIENTES and is filtered out of B├üSICOS to avoid dupes). To extend: add more `add(...)` rows with an `en:` alias.

The big `BASE_FOODS` IIFE is hard to edit by exact-match; to regenerate it, write the new block to a temp file and splice with Node: `h.replace(/const BASE_FOODS=\(function\(\)\{[\s\S]*?\n\}\)\(\);/, newBlock)` (verify exactly one match first).

**`openFoodAdd(preset, lockTag)`** ŌĆö the food-add sheet. Layout (topŌåÆbottom): optional COMIDA tag field + chips (a `#fa_tagtxt` input; **hidden** when `lockTag`) ŌåÆ `.fa-top` (search `#fa_q` + scan/search/register buttons, icon-over-label cards, equal width) ŌåÆ `.fa-listwrap` (RECIENTES/RESULTADOS list, fills + scrolls). `curTag()` reads `#fa_tagtxt` (works hidden). Opened from the **+ menu** (`addmeal`, editable tag) or from a **meal summary** (`addtomeal` ŌåÆ `openFoodAdd(tag, true)` ŌåÆ titled `add food ┬Ę <tag>`, no tag editor, straight to the list).

### External (client-side, no keys/backend)
- **OpenFoodFacts** ŌĆö barcode lookup (`/api/v2/product/{code}.json`) + name search (`/cgi/search.pl`). Barcode-not-found ŌåÆ offers **photo-of-label OCR** (primary) and name search (both link the food to the scanned code in `foods[]`).
- **Live barcode scanner** (`openBarcode()` ŌåÆ `startScan()`) ŌĆö MFP-style: tap scan ŌåÆ camera opens immediately with center reticle overlay ŌåÆ auto-detects ŌåÆ foundŌåÆ`openVerify` (forward), not-foundŌåÆ`openRegisterMenu(code)` (register, barcode linked). **Architecture = live preview + background still-frame decode** (the key fix ŌĆö Quagga LiveStream and `BarcodeDetector` both fail on iOS Safari): we own the `<video>` via `getUserMedia` (`facingMode:environment` 1280├Ś720), then a loop every ~360 ms grabs the current frame to an offscreen canvas (center band, ~62% height, downscaled Ōēż1000 px) and decodes it **as a still image** ŌĆö `BarcodeDetector.detect(canvas)` if available (Android/desktop, faster ~170 ms), else `Quagga.decodeSingle({src:canvas.toDataURL()})` (reliable on iOS). A `decoding` mutex serializes decodes so the loop and the manual button never run Quagga concurrently. **On hit:** `navigator.vibrate(55)` + green lock-on animation (`.scan-reticle.hit` ŌåÆ green frame `bcpop` + `Ō£ō` `bcchk`, scan line hidden) so the user can lower the phone, ~430 ms pause, then `doLookup`. **­¤ōĘ capturar** button = thorough full-res decode of the *current preview frame* via `decodeImgSrc()` (multi-patch `decodeSingle`), or a file photo (`decodeBarcodeImage`) if no live camera ŌĆö no longer ignored, since there's no running LiveStream to collide with. Manual numeric entry is the last fallback. `window._scanner={stream,video,timer}`; `stopScan()`/`closeModal()` clear the timer + stop tracks so the camera light goes off on close. **Replaced html5-qrcode (v5 ZXing ŌåÆ v6 Quagga2 LiveStream ŌåÆ v47 still-frame decode)** because live detection never worked reliably on iPhone.
- **Tesseract.js** (CDN, lazy) ŌĆö photoŌåÆOCR of nutrition labels (FALLBACK when AI proxy off). `prepImage()` preps image; `parseNutritionText()` handles kcal-vs-kJ, comma decimals, gŌåömg.
- **AI label reading (Gemini via Cloudflare Worker)** ŌĆö `proxy/gymtrk-ai-worker.js` holds the OWNER's one Gemini key (free tier) so users need no key. App: `AI_PROXY_URL` const (empty ŌćÆ falls back to OCR), `aiPhoto()` POSTs a compressed JPEG dataURL ŌåÆ proxy ŌåÆ structured nutrition ŌåÆ verify; `labelPhoto()` routes to AI if configured else OCR; `fileToDataURL/sanitizePer/labelFallback`. To enable: deploy the Worker (see `proxy/README.md`), set `AI_PROXY_URL`. Verified via mocked proxy in preview. **NOTE: the AI schema returns the OLD per-serving shape and will be upgraded together with the canonical food-model refactor (below).**
